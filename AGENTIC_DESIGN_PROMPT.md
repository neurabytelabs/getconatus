## L3 â€” Governance & Safety

### Agentic Design Principles

1. **Machine-readable soul:** The site should have semantic HTML + JSON-LD so agents crawling it can literally parse what Conatus is
2. **No dark patterns:** No popups, no email gates, no "sign up to see more." The philosophy is free. The tool is free. The soul is free.
3. **Terminal-first:** Every interactive element should feel like it could exist in a CLI. The web version is a beautiful wrapper around terminal truth.
4. **Respect the operator:** The human configuring their agent is technical. Don't explain what an API is. Show the command. They'll understand.

### Performance Constraints

- **First Meaningful Paint:** < 800ms (the cursor must blink fast)
- **Total bundle:** < 150KB JS (radical minimalism)
- **No frameworks for the sake of frameworks.** If vanilla JS + CSS can do it, prefer that. React only if component complexity demands it.
- **Zero external requests on load** â€” no analytics, no fonts CDN, no tracking pixels. Self-contained.
- **Lighthouse:** 99+ across all metrics

### Content Rules

- Maximum 50 words per section (above the fold). Every word earns its place.
- All Spinoza quotes real, with Ethics reference
- Technical examples must be copy-paste functional
- No emoji in body text (only in section annotations/dev docs). The design does the talking.
- Latin terms: always with translation, but Latin FIRST (this is a philosophy site)
